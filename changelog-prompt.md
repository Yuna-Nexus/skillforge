# 版本更新执行指南

> 当用户告知新版本更新内容时，按以下步骤执行完整更新

---

## 第一步：了解当前文档结构

当前仓库包含以下文件：

```
SkillForge/
├── my-skills/                    # 个人原创技能目录
│   ├── article-writing/          # SCI论文写作技能
│   └── research-publication-strategy/  # 科研发表策略技能
├── skills-index.md               # 全网技能导航索引
├── VERSION.md                    # 版本管理文档
├── changelog-prompt.md           # 本文件（版本更新指南）
├── .gitignore                    # Git忽略配置
└── README.md                     # 项目入口文档
```

---

## 第二步：分析用户提供的更新信息

从用户输入中提取以下信息：

- [ ] 新版本号（如 v1.1.0）
- [ ] 更新类型（主要/次要/补丁）
- [ ] 新增功能列表
- [ ] 功能改进列表
- [ ] 问题修复列表
- [ ] 性能优化列表
- [ ] 其他变更说明

---

## 第三步：执行更新任务（按顺序）

### 3.1 更新 VERSION.md

**文件位置**: `d:\yuna_github\SkillForge\VERSION.md`

**需要修改的部分**:
1. 更新"当前版本"表格中的版本号和日期
2. 在"版本历史"部分新增版本条目
3. 按格式填写更新内容分类

**操作步骤**:
1. 读取 VERSION.md 文件
2. 将旧版本信息移动到"版本历史"
3. 更新"当前版本"为最新信息
4. 添加新版本的历史记录

---

### 3.2 更新 README.md

**文件位置**: `d:\yuna_github\SkillForge\README.md`

**当前结构**:
- 标题和定位说明
- 我的原创（技能列表表格）
- 全网技能导航（链接到 skills-index.md）
- 目录结构说明

**需要检查/修改的部分**:
1. 检查"我的原创"部分是否需要添加新技能
2. 检查"目录结构"是否需要更新
3. 如有重大变更，添加版本说明段落

**操作步骤**:
1. 读取 README.md 文件
2. 检查技能列表是否需要新增条目
3. 检查目录结构说明是否需要更新
4. 保存修改

---

### 3.3 更新 skills-index.md（如需要）

**文件位置**: `d:\yuna_github\SkillForge\skills-index.md`

**当前结构**:
- 第一部分：AI Skill 平台与资源汇总表格
- 第二部分：推荐技能列表（按星级分类）

**需要修改的情况**:

**操作步骤**:
1. 读取 skills-index.md 文件
2. 如有新平台，添加到第一部分表格

4. 保存修改

## 示例执行流程

**用户输入**:
```
新版本 v1.1.0
- 新增 MCP 技能集成模块
- 修复了文档链接错误
- 在 skills-index.md 添加了新的推荐技能
```

**你的执行步骤**:

1. 读取 VERSION.md → 更新版本号为 v1.1.0，添加历史记录
2. 读取 README.md → 检查是否需要更新目录结构说明
3. 读取 skills-index.md → 添加新的推荐技能条目
4. 检查 my-skills/ → 确认是否有新增 MCP 相关技能目录
5. 生成更新摘要报告

## 注意事项

1. **先读取再修改**: 每次修改前必须先读取文件当前内容
2. **保持格式一致**: 遵循原有文档的格式和风格
3. **链接检查**: 确保 README.md 中的链接指向正确的文件
4. **逐步确认**: 每个文件修改后确认保存成功
